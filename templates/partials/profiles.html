Profiles

An extension of {{ this.title }} may be formalised as a profile. A profile is a {{ this.title }} which extends the default specification towards more specific needs.

A profile is declared on the profile property. For the default {{ this.title }} descriptor, this SHOULD be present with a value of default, but if not, the absence of a profile is equivalent to setting "profile": "default".

Custom profiles MUST have a profile property, where the value is a unique identifier for that profile. This unique identifier can be in one of two forms. It can be an id from the official Data Package Schema Registry, or, a URI that points directly to a JSON Schema that can be used to validate the profile.
